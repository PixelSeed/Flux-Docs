    常见问题¶

    1. 工具盒打不开或没有显示¶

    请按这个顺序排查：

    1. 是否执行过 floating_toolbox/install.py
    2. 是否重启过 Maya
    3. 共享 Documents/maya/modules 目录下是否生成了 floating_toolbox.mod
    4. 在 Script Editor 中直接执行 from floating_toolbox import show_toolbox; show_toolbox() 是否有报错

    如果仍然失败，优先查看 Maya Script Editor 的异常栈。

    2. 打开后语言不是我想要的¶

    请在设置页切换语言并保存，然后：

    1. 关闭并重新打开工具盒
    2. 必要时重启 Maya

    当前语言资源文件位于：

    • floating_toolbox/locales/zh_CN.json
    • floating_toolbox/locales/en_US.json

    3. Full Panel 快捷键没有生效¶

    请检查：

    1. 设置页里是否真的保存了快捷键
    2. 该快捷键是否和 Maya 现有热键冲突
    3. 你是否在保存后重新触发了工具盒或相关热键注册逻辑

    如果快捷键冲突，设置页会显示冲突提示。

    4. 我添加的外部工具没有显示¶

    当前外部工具发现依赖两件事：

    1. 额外工具路径已在设置页保存，或通过 FLUX_TOOL_PATHS 提供
    2. 包式工具目录中包含有效的 tool_manifest.json

    如果你修改了工具路径，通常建议重启 Maya 或至少重新打开工具盒。

    5. AI 面板无法使用¶

    请检查：

    1. 设置页中的 AI Enabled 是否开启
    2. 提供商、模型和 Base URL 是否有效
    3. API Key 是否已填入，或已通过环境变量提供

    当前支持的环境变量包括：

    • OPENAI_API_KEY
    • DEEPSEEK_API_KEY

    6. AI 没有调用我预期的工具¶

    先不要直接判断为“AI 坏了”，优先确认：

    1. 工具本身是否已经被工具盒发现并注册
    2. 工具 contract 是否有效
    3. 当前任务是否真的适合被编排成可执行步骤
    4. 是否需要先同步 AI 演示工具做链路验证

    7. 配置文件在哪¶

    工具盒当前会同时使用版本级和共享级配置目录。

    典型路径类似：

    Documents/maya/modules/floating_toolbox.mod
Documents/maya/<version>/prefs/floating_toolbox
Documents/maya/prefs/floating_toolbox_shared

    其中 .mod 是共享安装注册；用户可见并应跨 Maya 版本一致的数据优先保存在 floating_toolbox_shared，版本级目录主要保留运行态、日志、缓存或兼容迁移数据。

    更详细说明见 配置说明。

    8. 什么时候应该看 internal_docs¶

    如果你是普通使用者，通常不需要。

    只有在这些场景才建议看：

    • 你要接入新工具
    • 你要调试能力包协议
    • 你在参与架构维护
    • 你要追溯某个历史设计决策
