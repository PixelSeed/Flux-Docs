    架构说明¶

    Maya 悬浮工具盒采用 MVVM (Model-View-ViewModel) 架构模式，实现 UI 与业务逻辑的分离。

    核心模块详解¶

    1. ToolManager（工具管理器）¶

    职责：工具的注册、执行、搜索、收藏和持久化管理

    设计模式：单例模式

    from floating_toolbox.core.tool_manager import ToolManager

manager = ToolManager()  # 总是返回同一实例

    核心 API：

    方法 说明
    register_tool(tool_id, name, category, callback, ...) 注册工具
    get_tool(tool_id) -> Tool 获取工具实例
    get_all_tools() -> List[Tool] 获取所有工具
    get_tools_by_category(category) -> List[Tool] 按分类获取工具
    get_categories() -> List[str] 获取所有分类名
    search_tools(query) -> List[Tool] 搜索工具
    toggle_favorite(tool_id) 切换收藏状态
    toggle_pin(tool_id, category) 切换置顶状态
    get_favorite_tools() -> List[Tool] 获取收藏的工具
    get_recent_tools(limit) -> List[Tool] 获取最近使用
    save_config() 保存配置（节流版本）
    save_config_immediate() 立即保存（关键操作时）
    load_plugins() 加载所有插件
    reload_plugins() 热重载插件

    Tool 类属性：

    class Tool:
    id: str              # 唯一标识符
    name: str            # 显示名称
    category: str        # 所属分类
    subcategory: str     # 子分类（可选）
    callback: Callable   # 执行回调
    icon: str            # 图标路径
    description: str     # 描述文本
    is_favorite: bool    # 是否收藏
    pinned_categories: List[str]  # 在哪些分类置顶
    use_count: int       # 使用次数
    last_used: str       # 最后使用时间 (ISO格式)
    is_user_tool: bool   # 是否为用户创建
    code: str            # 用户工具代码
    language: str        # 代码语言 (Python/MEL)

    2. CategoryManager（分类管理器）¶

    职责：分类的 CRUD、排序、折叠状态和图标管理

    设计模式：单例模式

    核心 API：

    方法 说明
    add_category(name, icon, order) 添加分类
    get_category(name) -> Category 获取分类
    get_all_categories() -> List[Category] 获取所有分类（按顺序）
    get_category_names() -> List[str] 获取分类名列表
    rename_category(old, new) -> bool 重命名分类
    remove_category(name) -> List[str] 删除分类，返回工具ID列表
    set_category_icon(name, icon) 设置图标
    get_category_icon(name) -> str 获取图标路径
    set_category_order(order: List[str]) 设置分类顺序
    toggle_collapse(name) -> bool 切换折叠状态
    is_collapsed(name) -> bool 检查是否折叠
    get_tool_order(category) -> List[str] 获取工具排序
    set_tool_order(category, order) 设置工具排序
    get_subcategory_order(category) -> List[str] 获取子分类排序
    set_subcategory_order(category, order) 设置子分类排序

    Category 数据类：

    @dataclass
class Category:
    name: str           # 分类名称
    icon: str = ""      # 图标路径
    order: int = 0      # 排序权重
    is_collapsed: bool = False  # 折叠状态

    节流保存机制：

    # 500ms 内的多次调用只执行一次磁盘写入
category_manager.set_collapsed("Animation", True)  # 触发节流保存
category_manager.set_category_icon("Animation", "anim.svg")  # 合并到同一次保存

    3. ConfigManager（配置管理器）¶

    职责：JSON 配置文件的读写

    核心 API：

    方法 说明
    load_config() -> dict 加载主配置
    save_config(data: dict) 保存主配置
    load_user_tools() -> dict 加载用户工具
    save_user_tools(data: dict) 保存用户工具
    delete_user_tool(tool_id) 删除用户工具

    配置文件结构：

    "tool_id": {

      "pinned_categories": ["Animation"],

// categories.json

    4. ToolboxViewModel（视图模型）¶

    职责：连接 View 和 Model，处理业务逻辑

    设计模式：MVVM 中的 ViewModel

    核心信号：

    信号 触发时机
    tools_changed 工具列表变更
    favorites_changed 收藏状态变更
    recent_tools_changed 最近使用变更
    search_results_changed(query) 搜索结果变更
    categories_changed 分类变更

    核心方法：

    # 获取数据
viewmodel.all_tools          # 所有工具
viewmodel.categories         # 所有分类
viewmodel.favorite_tools     # 收藏的工具

# 用户操作
viewmodel.execute_tool(tool_id)       # 执行工具（带 Undo 支持）
viewmodel.toggle_favorite(tool_id)    # 切换收藏
viewmodel.toggle_pin(tool_id, cat)    # 切换置顶
viewmodel.search(query)               # 搜索

# 位置管理
viewmodel.get_position() -> (x, y)    # 获取面板位置
viewmodel.set_position(x, y)          # 保存面板位置

    FloatingBall（状态管理器）¶

    职责：管理三个面板的生命周期和切换

    ToolTreeModel（工具树模型）¶

    职责：为 QTreeView 提供层级数据

    结构：

    Root
├── 收藏 (Category)
│   ├── 工具A (Tool)
│   └── 工具B (Tool)
├── 建模 (Category)
│   ├── 基础 (Subcategory)
│   │   ├── 创建立方体 (Tool)
│   │   └── 创建球体 (Tool)
│   └── 变形 (Subcategory)
│       └── 平滑网格 (Tool)
└── 动画 (Category)
    └── 设置关键帧 (Tool)

    模块拆分：

    文件 职责
    tool_model.py 主模型，继承 QAbstractItemModel
    tool_tree_data.py ToolTreeDataBuilder - 树结构构建
    tool_tree_dnd.py ToolTreeDragDropHandler - 拖拽处理

    拖拽排序支持：

    • 分类拖拽：调整分类顺序
    • 子分类拖拽：同一分类内调整顺序
    • 工具拖拽：同一分类/子分类内调整顺序

    设计决策¶

    为什么使用 MVVM？¶

    优势 说明
    可测试性 ViewModel 和 Model 可脱离 Qt 独立测试
    可维护性 UI 变更不影响业务逻辑
    可扩展性 新增面板只需注入 ViewModel
    松耦合 通过信号通信，减少直接依赖

    为什么使用单例模式？¶

    类 原因
    ToolManager 确保全局只有一个工具注册表
    CategoryManager 统一管理分类状态
    ConfigManager 避免重复文件 I/O

    为什么使用节流保存？¶

    问题：用户快速连续操作（如批量折叠分类）会触发大量磁盘写入

    方案：500ms 延迟 + 合并写入

    def _save_config(self):
    if self._pending_save:
        return  # 已有待保存任务
    self._pending_save = True
    QTimer.singleShot(500, self._do_save_config)
