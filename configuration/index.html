    配置说明¶

    本文档只描述当前代码里已经存在并稳定保存的配置项，不讨论内部实验开关的全部细节。

    1. 设置页当前有哪些分类¶

    设置页目前包含这些栏目：

    • 常规
    • 分类栏
    • 工具路径
    • AI
    • 数据看板

    2. 常规设置¶

    常规页里最重要的几项包括：

    • 界面语言
    • Full Panel 缓存
    • Full Panel 快速切换动画
    • Interface 1 / Interface 3 切换快捷键
    • 启动时检查更新

    默认值来自当前代码中的 SettingsViewModel.reset_to_defaults()。

    3. 工具路径设置¶

    你可以通过设置页添加额外工具目录，工具盒会把这些目录当成外部包式工具根目录来扫描。

    当前扫描来源包括：

    1. 内置工具目录 floating_toolbox/tools
    2. 环境变量 FLUX_TOOL_PATHS
    3. 设置页中保存的 extra_tool_paths

    如果你修改了工具路径，建议重新打开工具盒；在多数场景下，重启 Maya 更稳妥。

    4. AI 设置¶

    当前设置页支持：

    • Provider
    • Model
    • Base URL
    • API Key

    当前内置支持的提供商：

    • OpenAI
    • 默认模型：gpt-4o-mini
    • 默认 Base URL：https://api.openai.com/v1
    • DeepSeek
    • 默认模型：deepseek-v4-pro
    • 默认 Base URL：https://api.deepseek.com

    如果你不想把 API Key 写进配置文件，也可以使用环境变量：

    • OPENAI_API_KEY
    • DEEPSEEK_API_KEY

    5. 配置文件保存在哪里¶

    工具盒当前会同时使用共享目录和版本级目录。

    跨版本共享目录¶

    用于保存用户可见、应在多个 Maya 版本之间保持一致的数据，例如：

    Documents/maya/prefs/floating_toolbox_shared

    这里通常包含：

    • toolbox_config.json
    • categories.json
    • marking_menu.json
    • user_tools/
    • ui_config.json
    • 标记菜单模板
    • 工作流模板

    版本级目录¶

    用于保存当前 Maya 版本下的运行态、日志、缓存或兼容迁移数据，例如：

    Documents/maya/2026/prefs/floating_toolbox

    这里通常会包含：

    • 使用统计
    • 执行日志
    • AI 日志
    • 导入审计日志
    • 其他运行期缓存

    6. 配置损坏时会发生什么¶

    当前 ConfigManager 具备基础恢复逻辑：

    • 写入时使用临时文件与替换写入
    • 主文件损坏时会尝试从 .bak 备份恢复
    • schema 版本过旧时会自动迁移到当前版本

    7. 哪些设置保存后立即生效¶

    通常可以立即生效或较快生效的：

    • Full Panel 热键
    • 面板缓存/动画相关设置

    通常建议重新打开工具盒后再观察的：

    • 语言切换
    • 工具路径变更
    • 某些 AI 相关配置

    8. 建议的最小配置¶

    如果你只是日常使用，推荐先完成这几项：

    1. 选择语言
    2. 设置 Full Panel 切换快捷键
    3. 根据需要开启 Full Panel 缓存
    4. 如果要用 AI，再填写 Provider / Model / API Key
