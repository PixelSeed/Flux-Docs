    安装指南¶

    本文档面向第一次安装 Maya Floating Toolbox 的用户，按当前仓库实现给出最直接的安装路径。

    环境要求¶

    • Maya 2017+
    • 可访问本仓库代码
    • 具备在 Maya Script Editor 中运行 Python 的能力

    推荐安装方式¶

    1. 获取仓库代码¶

    将仓库克隆或下载到本地，例如：

    D:\Pycharmpro\Maya2026_pro\MayaTool

    2. 在 Maya 中执行安装脚本¶

    打开 Maya 的 Script Editor，切到 Python 标签，执行：

    exec(open(r"D:\\Pycharmpro\\Maya2026_pro\\MayaTool\\floating_toolbox\\install.py", "r", encoding="utf-8").read())

    当前安装脚本会自动完成这些动作：

    • 检测本机用户目录下已存在的 Maya 版本，用于兼容清理
    • 将 floating_toolbox.mod 写入共享 Documents/maya/modules 目录
    • 清理旧版按 Maya 版本写入的残留 .mod
    • 尝试创建 FloatingToolbox Shelf 和 Toolbox 按钮
    • 支持同一台机器上的多版本 Maya 共用同一份安装注册

    3. 重启 Maya¶

    重启后，模块路径、Shelf 按钮和相关初始化逻辑会更稳定。

    启动工具盒¶

    重启后，在 Script Editor 中执行：

    from floating_toolbox import show_toolbox
show_toolbox()

    也可以直接打开完整面板：

    from floating_toolbox import show_full_panel
show_full_panel()

    安装后建议立即检查¶

    1. 是否能正常打开工具盒主窗口。
    2. 是否自动创建了 FloatingToolbox Shelf。
    3. 设置页是否可正常打开。
    4. 如果你依赖 Full Panel 快捷键，是否可以在设置中成功保存并生效。

    手动验证模块文件¶

    安装脚本会把 .mod 文件写到 Maya 用户共享 modules 目录。典型路径类似：

    Documents/maya/modules/floating_toolbox.mod

    如果工具盒无法加载，可以先确认这个文件是否存在，并确认内容指向当前仓库根目录。

    旧版路径 Documents/maya/<version>/modules/floating_toolbox.mod 只作为兼容清理对象，不再作为默认安装目标。

    数据路径说明¶

    用户可见、应跨 Maya 版本保持一致的数据默认保存在：

    Documents/maya/prefs/floating_toolbox_shared

    版本级目录只保留运行态、日志、缓存或兼容迁移数据：

    Documents/maya/<version>/prefs/floating_toolbox

    升级建议¶

    • 当前测试阶段，直接更新仓库代码后，可重新执行一次 floating_toolbox/install.py 刷新共享注册。
    • 正式发布包阶段，可使用 floating_toolbox/upgrade.py 处理工具盒本体升级；它不处理能力包升级。
    • 如果你升级了 Maya 主版本，通常不需要再为新版本单独写一份 .mod；共享 Documents/maya/modules 会被多个 Maya 版本读取。

    卸载建议¶

    卸载脚本默认只解除注册和入口，不删除用户数据：

    • 删除共享 Documents/maya/modules/floating_toolbox.mod
    • 同步清理旧版 Documents/maya/<version>/modules/floating_toolbox.mod
    • 移除 FloatingToolbox Shelf 入口
    • 默认保留 Documents/maya/prefs/floating_toolbox_shared 和版本级 prefs 数据

    只有用户在卸载时明确确认，才会删除配置、缓存和本地偏好。
